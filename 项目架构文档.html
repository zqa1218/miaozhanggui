喵掌柜自助下单系统面向摄影/妆娘/棚主的预约接单平台 · v1.0

一、系统概览

喵掌柜是一套面向小型摄影/妆娘/棚主的多角色预约接单平台。商家（摄影师、妆娘、棚主）通过 B端管理后台创建项目、配置时间轴与定价，客户通过 C端浏览项目、选择服务时段并支付定金完成预约。

核心能力： 项目上架 → 时间轴配置 → 客户预约 → 定金锁定 → 商家确认 → 服务完成 → 尾款结清

4.2 请求处理流程

HTTP Request │ ├── Middleware Chain ────────────────── │ helmet() → cors() → morgan() → rateLimiter → deviceId │ → auth/optionalClientAuth → validator(Joi) ── │ ▼ Controller ← 解析 req.query / req.body / req.user │ ▼ Service ← 业务逻辑 + 事务编排 + DTO 映射 │ ▼ Repository ← knex SQL 查询 │ ▼ MySQL / Redis

每个模块遵循 routes → controller → service → repository 分层，其中：

  • Routes： 定义 HTTP 方法、路径、中间件链
  • Controller： 解析请求、调用 service、通过 res.rh 统一响应
  • Service： 核心业务逻辑、事务编排、权限校验
  • Repository： 纯 SQL 查询封装，不包含业务逻辑
  • Validator： Joi schema 定义，在路由层通过中间件执行

4.4 共享基础设施

统一响应格式 (res.rh)

res.rh.success(data, msg) → { success: true, code: 0, data, message }

res.rh.fail(msg, status) → { success: false, code: 1, message }

res.rh.error(msg) → { success: false, code: 1, message } + 500

错误体系 (AppError + errorCodes)

所有业务异常抛出 new AppError(ERROR_CODES.STUDIO_NOT_FOUND, 404)，由 errorHandler 中间件统一捕获。分为 isOperational=true（预期内）和 unexpected 两种。

WebSocket 实时通知

订单创建后，通过 WebSocket 向对应 m_id 的商家推送通知，B端实时感知新订单。

双锁时段机制

pre_lock（第一重）：客户支付定金后自动创建，标记该时段不可被其他客户预约。
hard_lock（第二重）：商家确认收款后升级，正式排入工作日历。

5.3 API 层

七、部署运维

一键启动 (start.sh)

  1. docker compose up -d（MySQL + Redis）
  2. npm install（后端 + 前端依赖）
  3. npx knex migrate:latest（数据库迁移）
  4. node src/server.js（后端 :3000）
  5. npx vite（前端 :5173）

生产部署 (PM2)

npm run prod → pm2 start ecosystem.config.js，以 Cluster 模式运行，支持零停机 reload。

数据库迁移

当前最新迁移 038_add_address_fields.js（studios 增 address_required，orders 增 address/address_valid）。迁移脚本均为幂等设计（先检查列是否存在）。
