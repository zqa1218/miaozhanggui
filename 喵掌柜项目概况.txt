================================================================================
                      喵掌柜自助下单系统 —— 项目概况
================================================================================

一、项目概述
--------------------------------------------------------------------------------
"喵掌柜"是一个面向摄影/写真行业的在线预约下单平台，支持**双端架构**：

  ● C端（客户端）  — 普通用户浏览摄影项目、选择时段、在线下单预约
  ● B端（管理后台）— 商户管理项目、查看排期、处理订单、确认收款

项目采用前后端分离的 SPA 架构，部署在单台服务器上。C端和B端共用同一个后端 API 服务，
前端编译为静态文件后由 Express 统一托管。

核心业务流程：
  用户浏览工作室 → 选择摄影项目 → 挑选日期和时段 → 填写信息下单
  → 微信扫码付定金 → 商户后台确认收款 → 锁定时段（hard_lock）
  → 拍摄完成 → 付尾款 → 订单完成

四、后端架构详解
--------------------------------------------------------------------------------

【架构模式】三层架构 + 模块化
  每个业务模块内部遵循统一模式：
    routes.js     → 路由定义 + 中间件链
    controller.js → 请求处理（薄层，只做参数提取和响应返回）
    service.js    → 业务逻辑（核心，可跨模块调用）
    repository.js → 数据库访问（SQL 操作封装）
    validator.js  → Joi 校验规则

【核心设计模式】

  1. 统一响应格式
     所有 API 返回 { success: boolean, data: any, message: string }
     通过 res.rh.success() / res.rh.fail() / res.rh.error() 统一输出

  2. 业务错误类 (AppError)
     携带错误码 + HTTP 状态码 + isOperational 标记
     全局错误处理中间件区分 业务错误(4xx) / 未知错误(5xx)

  3. 二重锁机制（防超卖核心）
     第一重 pre_lock：用户支付定金后自动锁定时段（软锁）
     第二重 hard_lock：商户后台确认后转为硬锁（只有 hard_lock 才真正占位）
     配合 timeCollisionEngine 的行级锁 (SELECT FOR UPDATE) 防止并发冲突

  4. 价格计算引擎 (priceEngine.js)
     纯函数，零副作用，支持四种计费模式：
       time       — 按时段时长计费
       photo      — 按拍摄张数计费
       photo_time — 按张数+时段计费
       包天        — 固定包天价格
     功能：节假日溢价 / 多时段阶梯折扣 / 额外人数费 / 定金比例

  5. 时段碰撞检测引擎 (timeCollisionEngine.js)
     计算总服务时长 = 基础拍摄 + 模特加时 + 间隔休息
     闭合时间段重叠检测
     支持事务内行级锁（防止并发插入重叠预订）

  6. 状态机（订单生命周期）
     待支付 → 已付定金 → 尾款待确认 → 已结清 → 已完成拍摄
     任意状态可发起：改期申请 / 取消申请 → 审批 → 执行
     支持：恢复已取消订单 / 永久删除已取消订单


五、前端架构详解
--------------------------------------------------------------------------------

【设计原则】
  ● 适配器模式：所有 HTTP 请求通过 utils/http.js 发出，API 层不直接依赖 axios，
    便于未来移植到微信小程序（替换为 wx.request）
  ● 存储抽象层：所有本地持久化通过 utils/storage.js，可一键切换后端存储
  ● 组件/页面分离：views/ 存放路由页面（厚组件），components/ 存放可复用组件
  ● 业务逻辑外置：composables/ 存放可复用的响应式逻辑

【路由设计】
  双端路由，两个主布局：

  C端（ClientLayout）：
    /                         → 重定向到 /studios
    /studios                  → 工作室列表
    /studios/:id              → 工作室详情
    /styles                   → 样式列表
    /booking/:studioId        → 下单步骤1
    /booking/:studioId/step2  → 下单步骤2
    /my-orders                → 我的订单
    /my-orders/:id            → 订单详情
    /login                    → C端登录（独立布局）

  B端（AdminLayout）：
    /admin                    → 重定向到 /admin/orders
    /admin/login              → 管理端登录
    /admin/orders             → 订单管理 ★
    /admin/studios            → 项目管理
    /admin/styles             → 预设管理
    /admin/studio/create/*    → 三步创建项目
    /admin/studio/edit/:id    → 编辑项目
    /admin/schedule           → 排期管理
    /admin/settings           → 设置
    /admin/notifications      → 通知
    /admin/logs               → 操作日志
    /admin/dashboard          → 超级管理员面板

  全局导航守卫：
    ● B端需要鉴权的路由 → 无 Token 跳转到 /admin/login
    ● C端需要登录的路由 → 无 Token 跳转到 /login?redirect=xxx
    ● B端已登录访问登录页 → 直接跳转到 /admin/orders

【状态管理 (Pinia Stores)】

  auth.js          — 管理端认证状态（token, mId, shopName）
  clientAuth.js    — C端认证状态（OAuth + JWT Token）
  booking.js       — 预约流程（可用时段、锁状态、联系方式）
  order.js         — 订单管理（列表、筛选、统计）
  schedule.js      — 排期管理（日历、已预订、不可用时段）
  studio.js        — 摄影项目列表
  studioCreate.js  — 项目创建向导（多步表单状态）
  style.js         — 预设样式管理
  settings.js      — 商户设置
  wizardB.js       — B端多步向导
  wizardC.js       — C端下单向导

【UI 组件设计】
  ● Element Plus 作为基础组件库
  ● 自定义组件：
    CalendarGrid.vue            — B端排期日历（可拖选不可用时段）
    CalendarPicker.vue          — C端日期选择器
    FlexibleTimelinePicker.vue  — 灵活时段选择（支持多个不连续时段）
    StepIndicator.vue           — 步骤进度条
    DeclarationDialog.vue       — 免责声明确认弹窗
    ExtraItemsEditor.vue        — 附加项目编辑器（动态增删）

    RefreshButton.vue           — 全局刷新按钮（防抖+loading）

七、部署与运维
--------------------------------------------------------------------------------

【部署流程】(deploy.sh)
  1. git pull origin main             拉取最新代码
  2. npm ci                           安装后端依赖
  3. npx knex migrate:latest          执行数据库迁移（production 环境）
  4. cd frontend && npm ci && build   安装前端依赖并构建
  5. pm2 reload ecosystem.config.js   零停机重启服务

【PM2 配置】
  ● 进程名：miaozhanggui-api
  ● 模式：cluster（多核利用）
  ● 实例数：max（自动检测 CPU 核数）
  ● 零停机：wait_ready + listen_timeout
  ● 日志：./logs/pm2-error.log / ./logs/pm2-out.log

【环境变量】
  服务器配置    — NODE_ENV, PORT, HOST
  MySQL         — MYSQL_HOST, MYSQL_PORT, MYSQL_USER, MYSQL_PASSWORD, MYSQL_DATABASE
  Redis         — REDIS_HOST, REDIS_PORT, REDIS_PASSWORD（可选）
  JWT           — JWT_SECRET, JWT_EXPIRES_IN, JWT_REFRESH_EXPIRES_IN
  文件上传      — UPLOAD_DIR, MAX_FILE_SIZE
  CORS          — CORS_ORIGINS（逗号分隔白名单）
  微信          — WECHAT_APPID, WECHAT_SECRET, WECHAT_MCHID, WECHAT_API_KEY

【启动命令】
  ● 开发：npm run dev (nodemon 热重载)
  ● 生产：npm run prod (PM2 cluster)
  ● 迁移：npm run migrate
  ● 前端开发：cd frontend && npm run dev (Vite HMR, 端口 5173, 代理 /api → 3000)


八、关键特性总结
--------------------------------------------------------------------------------

  1. 双端架构        — C端（用户下单）+ B端（商户管理），共享后端 API
  2. 订单状态机      — 完整生命周期管理（待支付→已付定金→已结清→已完成）
  3. 二重锁机制      — 防超卖的 pre_lock + hard_lock 双层时段锁定
  4. 时段碰撞检测    — 闭合区间重叠检测 + 数据库行级锁(SELECT FOR UPDATE)
  5. 多模式计价      — time/photo/photo_time/包天，含节假日+阶梯折扣
  6. 改期/取消审批   — C端发起申请 → B端审批 → 自动更新时段
  7. Excel 导入导出  — 订单和排期支持批量导入导出
  8. WebSocket 推送  — 订单状态变更实时通知商家和用户
  9. 邀请码注册      — 关闭公开注册，仅通过邀请码创建商户
  10. OAuth 登录     — C端支持微信/QQ第三方登录
  11. 适配器模式     — 前端 HTTP/Storage 层抽象，便于移植小程序
  12. 零停机部署     — PM2 cluster + reload 实现无缝更新
