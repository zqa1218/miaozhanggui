━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1.1 商家配置初始化流 (Merchant Initialization Lifecycle)
  【B端 窗口 1】输入：项目基本信息 + 城市（`city`必填） + 开放接单公历日期数组（`availableDates[]`）。
  【B端 窗口 2】输入：一日连续总可接单范围（`baseStartTime` 至 `baseEndTime`） ──► 输入休息分段数 $N$ ──► 动态呈现 $N$ 行起止时间选择器录入具体的休息排除段 `restSlots: [{start_time, end_time}]` ──► 录入单场交替转场清理时间（`intervalRestTime`） ──► 勾选一键复刻。
  【B端 窗口 3】输入：勾选绑定已在 Window 0 创建好的样式资产 ID 集合（`selectedStyleIds[]`），或录入原生无样式单张/套餐价格 ──► 录入单张/套餐标准拍摄耗时、新人单张/套餐额外加时参数、定金比例。
  落库响应：后端启动隔离事务，将主表、打散的日期从表、多段休息时段表一次性全量持久化。

1.2 C端客户高精度精算询价流 (Client Precision Estimation Flow)
  客户进入项目 ──► 系统读取 `is_style_enabled` 状态进行 SKU 渲染分流（强约束勾选独立样式卡片） ──► 客户点击可预约日期。
  客户选择特定的“拍摄起始时间点” `bookingStartTime`（如 14:15）并勾选模特经验身份（新人/老手）。
  前端精算引擎自动触发（Headless 运算）：
    ① 提取耗时源：`baseShotTime` = 选定样式（或项目原生）对应的单张或套餐耗时（分钟）。
    ② 提取加时源：若为新人，`modelAddTime` = 对应的单张/套餐新人加时。
    ③ 计算闭合占用总时长：`TotalDuration = baseShotTime + modelAddTime + intervalRestTime`。
    ④ 推算绝对闭合终点：`bookingEndTime = bookingStartTime + TotalDuration`（分钟级进位换算）。
  视图响应：毛玻璃看板以微弱呼吸灯特效动态公示：【您的预计拍摄时间段为：14:15 至 15:30（已包含场次整理）】。

1.3 并发碰撞拦截与双锁死流 (Concurrency Collision & Double-Locking)
  【请求拦截预检】：前端自动将选定的闭合区间 `[bookingStartTime, bookingEndTime]` 发送至后端。后端使用区间交集碰撞算法进行双重交叉核对：
    - 检查一：当前选择区间是否与数据库该项目下已被他人占用的 `slot_bookings` 发生重叠（条件：开始 A < 结束 B 且 结束 A > 开始 B）。
    - 检查二：当前选择区间是否撞上了商家的 `studio_rest_slots` 固定休息不接单时段。
    - 结果：若发生任意重叠，前端闪红报警并强行将【下一步】按钮设为 `disabled`；若完全可用，绿标放行进入窗口 5。
  【第一重锁（定金预锁 `pre_lock`）】：客户在窗口 5 输入必填项 `roleName`（想要拍的角色名称）并提交。在线下扫码完成转账后，点击“我已支付完成”。触发 `/api/pay-deposit`。后端获取行级锁（`SELECT ... FOR UPDATE`），重做碰撞校验。通过后立即向 `slot_bookings` 插入当前闭合时间块，赋予 `lock_type = 'pre_lock'`。此时该时段在公共 C 端立刻置灰锁死，防御并发抢单。
  【第二重锁（摄影师接单锁 `hard_lock`）】：B 端摄影师在后台审查该角色订单的精算时段无误，手动点击 [确认接单]，触发 `/api/order/confirm-lock` 接口。后端在事务内将该时间块的锁状态升级为彻底锁死的 `lock_type = 'hard_lock'`，正式加入摄影师的订单工作表。

2.1 升级版 V3 数据库 Schema 微调映射
  - `studios` (项目主表增量字段)：
    * `city` (VARCHAR, 城市必填)
    * `base_start_time` (TIME, 每日接单起始，默认 "09:00")
    * `base_end_time` (TIME, 每日接单结束，默认 "21:00")
    * `interval_rest_time` (INT, 每单间隔休息分钟数)
    * `is_experience_enabled` (BOOLEAN, 是否开启经验选择)
    * `novice_single_add_time` / `novice_package_add_time` (INT, 新人加时分钟数)
    * `single_shot_time` / `package_time` (INT, 标准拍摄耗时分钟数)
  - `studio_rest_slots` (商家多段固定休息时段表 - 全新引入)：
    * `id` (INT, 主键) | `studio_id` (INT, 外键) | `start_time` (TIME) | `end_time` (TIME)
  - `slot_bookings` (高精度闭合时段占用表 - 字段级重塑)：
    * 废除旧的 V1 离散字符串列，重构为：`start_time` (TIME) | `end_time` (TIME) | `lock_type` (ENUM: 'pre_lock', 'hard_lock')
  - `orders` (订单主表增量字段)：
    * `role_name` (VARCHAR, 拍摄角色名称，强约束非空)

2.2 后端核心算法函数设计 (Backend Core Collision Engine)
  - `calcTotalDuration(baseShotTime, modelAddTime, intervalRest)`
    * 职责：计算包含转场转场休息在内的全局占用总分钟数。
  - `intervalsOverlap(aStart, aEnd, bStart, bEnd)`
    * 职责：标准区间重叠判定算法。返回 `(aStart < bEnd && aEnd > bStart)` 的布尔值，用于高精度碰撞拦截。
  - `checkTimeCollisionV3(mId, studioId, date, startTime, endTime, excludeOrderNo)`
    * 职责：事务内核心函数。结合 `FOR UPDATE` 行级锁，遍历 `studio_rest_slots` 与 `slot_bookings`，若发生 overlap 立即向业务层阻断抛出异常。
